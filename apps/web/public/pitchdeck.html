The Problem
Why SMBs are underserved by existing HR tools

63 million MSMEs in India employ over 110 million people — yet fewer than 5% use any form of structured HR software.

📊

Spreadsheet chaos

Leave tracking, attendance, and employee records are managed across disconnected Excel files shared over WhatsApp and email — error-prone, insecure, and impossible to audit.

💸

Enterprise HR software is too expensive

Tools like SAP SuccessFactors, Workday, and even BambooHR are priced for large enterprises — ₹1,500–₹3,000+ per employee/month — far beyond what a 30-person SMB can justify.

🧩

Fragmented point solutions

Businesses stitch together separate tools for leave, payroll, documents, and reviews — creating data silos, duplicate work, and no single source of truth for HR operations.

👤

No dedicated HR hire

SMBs cannot afford a full-time HR manager. The business owner or an office admin handles all HR tasks on top of their primary responsibilities — leading to non-compliance and poor employee experience.

Multi-Tenancy Model

Every table carries an org_id foreign key. The admin Supabase client (service role key) is used exclusively in server actions, bypassing RLS while maintaining org-level isolation via query filters.

Zero-API Architecture

All data mutations use Next.js Server Actions — no REST or GraphQL API layer needed. This eliminates an entire network round-trip and simplifies the codebase significantly.

Security & Access Control
Enterprise-grade RBAC built from the ground up

Four-Tier Role Hierarchy

👑
Owner
Full access to everything including billing
🛡️
Admin
All operations except billing ownership transfer
👔
Manager
View team data, approve/reject leaves, run reviews
👤
Employee
Self-service — own data only, no admin controls

Two-Layer Permission Enforcement

Layer 1 — Server Actions (Security)

Every mutating server action calls getCurrentUser() and checks role before touching the database. Even if the UI is bypassed, the server rejects unauthorised requests.

  • Add/edit/terminate employee → Admin only
  • Approve/reject leave → Manager+
  • Upload/delete documents → Admin only
  • Create review cycles, courses → Admin only

Layer 2 — UI Guards (UX)

Components receive the role prop from server pages and conditionally render admin-only buttons, dialogs, and action menus. Employees never see controls they can't use.

Data Isolation Architecture

org_id on every table

All 13 database tables carry an org_id FK. Cross-tenant data leakage is architecturally impossible — no query can return another org's data.

Supabase RLS enabled

Row Level Security is enabled on all tables as a secondary defence layer, even though server actions already filter by org_id.

Signed storage URLs

Document files are never publicly accessible. All downloads use 1-hour signed URLs generated server-side — expiring automatically after use.

Key Design Decisions

JSONB for flexible data

OKR items and review goals stored as JSONB — supports arbitrary structure without schema migrations as requirements evolve.

Soft deletes everywhere

Employees are marked "terminated" rather than deleted, preserving all historical data for leave records, reviews, and audit trails.

Auto-updated timestamps

Postgres triggers automatically update updated_at on key tables — no application-layer timestamp management required.

Upsert-safe acknowledgments

Document acknowledgment uses a unique constraint on (document_id, employee_id) with upsert — clicking Acknowledge twice is always safe.

Indexed for performance

All foreign keys, org_id columns, and status fields are indexed. Query performance remains constant as data grows to millions of rows.

Product Roadmap & Future Possibilities
Where JambaHR is going — near-term and long-term

Phase 2 — Next 3 Months

Plan-Based Feature Gating

Lock Reviews, Training, and Compliance behind Growth/Business tiers. Upgrade prompts for Starter users who try to access paid features.

Payroll & Compensation Module

Attendance & Time Tracking

Clock in/out, overtime calculation, late arrivals, half-day marks, and monthly attendance reports. Integrates with leave balances automatically.

Background Job Infrastructure

Trigger.dev or Inngest for scheduled jobs — leave approval reminders, training deadline alerts, compliance deadline warnings, payroll processing triggers.

Mobile-Optimised Experience

Responsive improvements for on-the-go managers — approve leave, view announcements, and check team status from mobile browsers without a native app.
